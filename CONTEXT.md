# JoJo Codex Pet

An unofficial, non-commercial fan project that turns named JoJo characters and Stands into installable Codex companions and presents them in a public vertical gallery.

## Language

**Fan Project**:
The complete unofficial, non-commercial JoJo-themed repository and gallery, offered free of charge and without advertising, with no claim of endorsement, affiliation, or authorization from the underlying rights holders.
_Avoid_: Official project, licensed project

**Ad-Free Gallery**:
The public Fan Project website, which carries no AdSense, display advertising, affiliate placement, or paid promotion.
_Avoid_: Ad-supported gallery, monetized fan site

**Temporary Production Gallery**:
The public, indexable Gallery served from `jojo-preview.easytry.shop` as the project's canonical site until a dedicated domain replaces it.
_Avoid_: Staging site, noindex preview

**Default Locale**:
The English version of every public Gallery route, paired with a user-selectable Simplified Chinese version of the same catalog content.
_Avoid_: Chinese-only site, automatic machine translation

**Project Code**:
Original software, scripts, schemas, and documentation authored for the Fan Project, separate from Character Assets and other franchise material.
_Avoid_: JoJo assets

**Split License**:
The repository boundary that places Project Code under MIT while excluding Character Assets and franchise-specific catalog content from that software license.
_Avoid_: Repository-wide MIT, single license

**Fan Asset Notice**:
The limited terms for Character Assets and franchise-specific catalog content, permitting personal non-commercial installation and use only to the extent contributors can grant rights and granting no rights in the underlying JoJo intellectual property.
_Avoid_: Open-source asset license, JoJo license

**Character Asset**:
A pet spritesheet, poster, preview, or related visual depicting a named JoJo character or Stand whose underlying intellectual property is owned by third parties.
_Avoid_: Open-source JoJo asset, licensed JoJo asset

**Chibi Pixel System**:
The Fan Project's unified visual language of compact proportions, bold dark contours, limited palettes, and iconic character or Stand silhouettes.
_Avoid_: Official JoJo style, traced sprite, mixed-media collection

**Free-Color Palette**:
A project-authored, Araki-inspired color composition chosen independently of a character's common anime or manga colors and never presented as canonical.
_Avoid_: Canonical palette, official colors

**Signature Palette**:
The single Free-Color Palette assigned to one Codex Pet and preserved across all of its animation states and look directions.
_Avoid_: Per-state recolor, palette pack

**Paired Palette**:
The relationship between the two distinct Signature Palettes in a Stand Pair, connected by a shared accent or deliberate complementary-color structure.
_Avoid_: Identical palette, unrelated palettes

**Codex Pet**:
One independently installable animated subject representing either a single JoJo character or a single Stand.
_Avoid_: Character-and-Stand combo, dual pet

**V2-Only Catalog**:
The compatibility boundary requiring every published or contributed Codex Pet to satisfy the Codex pet v2 spritesheet and metadata contract.
_Avoid_: V1 fallback, mixed-version catalog

**Repository Source Record**:
The canonical public GitHub repository URL recorded as the source for project-authored pets, without storing per-pet reference-image URLs.
_Avoid_: GitLab source, reference-image archive

**Catalog Pet ID**:
The stable Part-scoped identity of one Codex Pet, independent of its contributor, with a canonical variant or form qualifier when the Part and subject alone are not unique.
_Avoid_: Author-scoped ID, unscoped character name

**Official Installer**:
The project-owned, cross-platform installation path that resolves a Catalog Pet ID, verifies its runtime package, and writes it into the user's Codex pet directory.
_Avoid_: Manual-only install, unverified copy

**Single-Pet Install**:
The primary installation action, which installs exactly the selected Codex Pet and never implicitly adds its Stand Pair or Part Collection.
_Avoid_: Pair-by-default install, collection-by-default install

**Page View Count**:
The public aggregate number of times one canonical pet page is opened, displayed in the lower corner of that pet's Gallery card without identifying visitors or tracking installations, sessions, or in-page behavior.
_Avoid_: Install count, unique visitor, behavioral analytics

**User Feedback**:
Qualitative comments deliberately submitted by a user through the project's public feedback channel, separate from Page View Counts.
_Avoid_: Page view, inferred opinion

**State-Faithful Performance**:
A character-specific pose, expression, and motion treatment that preserves the interaction meaning of its assigned Codex animation state.
_Avoid_: Generic motion template, signature-move override

**Core Ability Cue**:
A packaged, pixel-art expression of one pet's defining ability, rendered as an opaque hard-edged effect connected to or overlapping the pet silhouette.
_Avoid_: Detached effect, full Stand summon

**Ability Beat**:
A state-specific variation of a Core Ability Cue whose action reinforces, rather than replaces, the meaning of that Codex animation state.
_Avoid_: Ability override, repeated attack loop

**Showcase Effect**:
A gallery-only visual layer that dramatizes a pet's ability without becoming part of the installable Character Asset.
_Avoid_: Pet sprite, Core Ability Cue

**Character Pet**:
A Codex Pet whose sole subject is one named JoJo character in one canonical Character Variant.
_Avoid_: Stand user package

**Character Variant**:
A canonical era or appearance of one character with a materially distinct identity or visual design that qualifies as its own Character Pet.
_Avoid_: Skin, animation state

**Character Lineage**:
The family of Character Pets representing the same individual across qualifying Parts, eras, or canonical appearances.
_Avoid_: One multi-era pet, costume set

**Stand Pet**:
A Codex Pet whose sole subject is one named Stand.
_Avoid_: Stand effect, character accessory

**Stand Pair**:
The canonical relationship linking one Character Pet with one specific Stand Pet from that character's Stand Lineage.
_Avoid_: Combined pet, paired sprite

**Stand Lineage**:
The ordered family of Stand Pets representing every qualifying Stand Form associated with one character.
_Avoid_: Multi-form pet, transformation row

**Stand Form**:
An officially named canonical stage with a persistent, materially distinct visual identity that qualifies as its own Stand Pet.
_Avoid_: Ability mode, temporary power state

**Ability Mode**:
A technique or power state that does not create a persistent, materially distinct Stand identity and therefore remains part of an existing Stand Pet.
_Avoid_: Stand Form, evolution

**Bonus Pet**:
An optional Codex Pet representing a visually autonomous sub-Stand or companion entity outside the core Stand Lineage.
_Avoid_: Required form, lineage stage

**Expansion Pet**:
An optional Character Pet or Stand Pet outside the Headline Roster, including allies, secondary antagonists, and non-headline Character Variants.
_Avoid_: Headliner, required pet

**Headline Roster**:
The cross-Part catalog limited to each Part's protagonist and Principal Antagonist, together with their qualifying Stand Lineages.
_Avoid_: Full cast, ensemble roster

**Principal Antagonist**:
The primary opposing character who drives a Part's central conflict, whether or not that character is the final combat opponent.
_Avoid_: Final opponent, final boss

**Part Collection**:
The canonical grouping of Headline Roster entries belonging to one numbered JoJo Part.
_Avoid_: Theme collection, release wave

**Release Wave**:
A quality-gated public delivery subset of the Headline Roster that may contain multiple Part Collections.
_Avoid_: Part Collection, complete roster

**Approval Gate**:
A required human review point covering the complete 36-pet roster before generation, the Pilot Set's base designs and final animations, or a later wave's QA overview and anomalies.
_Avoid_: Per-frame approval, unreviewed batch release

**Spoiler Gate**:
A user-controlled reveal that hides antagonist identities, evolved Stand Forms, and ability details behind a clear spoiler warning while leaving spoiler-safe browsing available.
_Avoid_: Spoiler-free catalog, unmarked reveal

**Contribution Gate**:
The milestone after the first public Release Wave passes quality acceptance, at which JoJo Community Submissions may open.
_Avoid_: Launch-day upload, account registration

**Community Submission**:
A GitHub pull request proposing a JoJo Expansion Pet for inclusion after the Contribution Gate.
_Avoid_: Direct upload, unrelated pet

**Pilot Set**:
The four Part 3 Headline Pets treated as an internal pre-release subset before the first Release Wave.
_Avoid_: Wave 0, public release

**Ongoing Part**:
A Part whose story has not concluded and whose Principal Antagonist position therefore remains deliberately unassigned.
_Avoid_: Incomplete collection, provisional villain
